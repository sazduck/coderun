Вам достался файл с логами запросов и ответов. К сожалению, файл повреждён: структурированные данные в формате JSON оказались перемешаны с произвольным мусорным текстом. Необходимо восстановить как можно больше данных.

Формально, дана строка S S. В ней могут встречаться корректные JSON-значения следующих типов:

    Объект: {"key": "value", "n": 1}
    Массив: [1, 2, "abc"]
    Строка: "hello"
    Целое число: 42, -7, 0

Гарантируется, что другие корректные JSON-типы (дробные числа, true, false, null) в данной задаче не используются.

Требуется выбрать такое множество непересекающихся подстрок S S, что:

    Каждая подстрока является корректным JSON-значением.
    Никакие две выбранные подстроки не пересекаются (то есть не имеют общих символов).
    Суммарная длина выбранных подстрок максимальна.

Формат ввода

Одна строка S S (длиной не более 50   000 50000 символов).

Формат вывода

В первой строке — два числа через пробел: количество выбранных JSON-значений k k и максимальная суммарная длина выбранных подстрок.
В каждой из следующих k k строк — два числа li​ и ri​ — границы соответствующего значения (начало и конец подстроки, конец невключительно, отсчёт от 0 0). Порядок строк не важен.


Примечание

В первом примере в строке есть три JSON-строки: "a" (позиции 6–9), "bbbbbbbbb" (позиции 8–19) и "c" (позиции 18–21). Строка "a" и строка "bbbbbbbbb" перекрываются на позиции 8, а "bbbbbbbbb" и "c" — на позиции 18. Оптимально взять "bbbbbbbbb" (длина 11).

Во втором примере выбраны объект {"key": "value"} (позиции 6–22, длина 16) и массив [1, 2, 3] (позиции 27–36, длина 9). Суммарная длина 25.

В третьем примере выбраны три объекта: {} (позиции 8–10, длина 2), {"x": 1} (позиции 11–19, длина 8) и {"y": [2, 3]} (позиции 24–37, длина 13). Суммарная длина 23.

Для работы с JSON можно использовать библиотеки:

    C++: #include <nlohmann/json.hpp> — https://github.com/nlohmann/json
    Kotlin: kotlinx.serialization.json — https://github.com/Kotlin/kotlinx.serialization
    Java: org.json.simple — https://github.com/fangyidong/json-simple

Если решаете на Java, публичный класс должен называться Solution.
